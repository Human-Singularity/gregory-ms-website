Installation

To embed the Gregory Observatory Widget in your website, follow these steps:

1. Include the required files in your HTML:

<!-- Widget CSS -->
<link rel="stylesheet" href="https://gregory-ms.com/js/gregory-observatory-widget.css">

<!-- Chart.js (required for visualization) -->
<script src="https://cdn.jsdelivr.net/npm/chart.js@4.4.0/dist/chart.umd.min.js"></script>

<!-- Widget JavaScript -->
<script src="https://gregory-ms.com/js/gregory-observatory-widget.js"></script>

2. Add a container element where you want the widget to appear:

<div id="gregory-observatory-widget"></div>

3. Initialize the widget with your configuration:

<script>
  // Initialize after page loads
  document.addEventListener('DOMContentLoaded', function() {
    GregoryObservatory.init({
      containerId: 'gregory-observatory-widget',
      categoryId: 30, // Autologous hematopoietic stem cell transplant
      apiUrl: 'https://api.gregory-ms.com',
      teamId: 1,
      subjectId: 1
    });
  });
</script>

Configuration Options

Option Type Required Default Description
containerId string Yes - ID of the HTML element where the widget will be rendered
categoryId number Yes* - ID of the category to display (e.g., 6 for Ocrelizumab)
categorySlug string Yes* - Slug of the category to display (e.g., 'ocrelizumab')
apiUrl string No 'https://api.gregory-ms.com' Base URL of the Gregory MS API
teamId number No 1 Team ID for filtering data
subjectId number No 1 Subject ID for filtering data (1 = Multiple Sclerosis)

* Either categoryId or categorySlug is required (not both)

Usage Examples

Example 1: Using Category ID

GregoryObservatory.init({
  containerId: 'my-widget',
  categoryId: 6  // Ocrelizumab
});

Example 2: Using Category Slug

GregoryObservatory.init({
  containerId: 'my-widget',
  categorySlug: 'natalizumab'
});

Example 3: Multiple Widgets on Same Page

<div id="widget-1"></div>
<div id="widget-2"></div>

<script>
  // First widget - Ocrelizumab
  GregoryObservatory.init({
    containerId: 'widget-1',
    categoryId: 6
  });

  // Note: For multiple widgets, you need to create separate instances
  // This requires modifying the widget code to support multiple instances
</script>

Features

  • No Build Process Required: Pure JavaScript, works without any build tools or frameworks
  • Responsive Design: Adapts to mobile, tablet, and desktop screens
  • Interactive Chart: Visualizes monthly research activity with Chart.js
  • Three Tabs:
    • Overview - Monthly chart showing articles and clinical trials
    • Articles - Latest research articles with ML relevance scores
    • Clinical Trials - Active and completed clinical trials
  • Lazy Loading: Data is fetched only when needed (on tab switch)
  • Pagination: "Load More" buttons for articles and trials
  • ML Predictions: Shows predictions from multiple machine learning models

Browser Support

The widget supports all modern browsers:

  • Chrome 90+
  • Firefox 88+
  • Safari 14+
  • Edge 90+
